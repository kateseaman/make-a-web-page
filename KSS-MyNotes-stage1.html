LESSON 1: basics of the web and html
The Internet

The internet is a collection of weg pages which are special documents written in HTML (hyper text mark up language). Those documents have a file extension of .htm and are readable by the various browsers. HTML documents are stored on servers, identified by a URL via http and read, interpreted and presented to the viewer by the browser on their computer. Only HTML files can be read by a browser. Coding must be written in the proper syntax for computers to understand them. Computers are stupid, they do exactly what they are told.

HTML Basics

HTML code uses elements and tags to define content and how it should be presented. Open-tags and close-tags define an element and elements can be nested within elements.

HTML elements can be either inline or block. Block elements have an invisible box around the content inside them, and the element consumes the entire space in that block so nothign else can be there. Inline elements consume just one line, but consume the entire line. An attribute is the property of an HTML element. Attributes provide reference to images, documents, URLs, etc.

DOM Structure

HTML uses a "tree structure" to organize content/elements where elements can be nested inside elements. Nested elements are noted in HTML code via indentation - more indentation denotes deeper nesting. In Google chrome, choosing tools/developer tools opens a panel to show structure of the page as a browser interprets it, in a "tree" format. DOM = Document Object Model.

A Note about "Elements"

An element can be text, a container or it can be blank. Webpages are made up of rectangles - either solitary or nested, and can be defined by a class name. The rectangles can be manipulated to appear to be any number of different shapes by using "layers" of styling to define background and foreground and make a rectangle appear circular for example.

Programming Text Editors

Text editing: Codepen.io is an online code editor good for small jobs and sharing code. Sublime text is downloadable, hard-drive-resident text editor and better for larger jobs. Sublime can be set to "auto complete" or predict what you will be typing. Once you type the first character of an open tag, it offers a predictive drop-down and once you select the proper tag, it will complete the closing tag for you.


LESSON 3: More Detail on CSS
CSS Basics

Stands for cascading style sheet and introduces formatting. CSS is a means to avoid unnecessary repetition in coding which saves everyone time and money, and improves website loading speed. You can have multiple CSS files referenced in HTML. The idea of "inheritance" is a key feature of CSS and defines how properties are applied to elements and "descendents."

Inheritance and Descendents
  • In general, descendent elements inherit text-related properties but but not box-related properties
  • Inheritance prevents repetition in code-writing, drives web pages to load faster and enables savings on bandwidth and development costs
  • Selectors define what elements of the page a style will apply to.
  • The syntax is: .description {attribute;value} where everything inside the curly brackets is a rule
Ways to include CSS in a Web Page
  1. Put in the head element in between style tags. Good for only small jobs
  2. link HTML file to a separate CSS file (good for larger projects
  • All HTML code is in one file .html
  • All CSS code is in a separate .css file
  • Add a head element to the HTML file and add a link tag inside the lead to link the two files together

Writing the "style code" inline with the HTML (bad idea = repeats a lot of code!)

Code Comments

Code comments are ignored by computers. Both HTML and CSS can include code comments so programmers can make notations to themselves or other programmers. Code comments in HTML start with a an open tag and exclamation "!" symbol, CSS comments start with an backslash + asterisk "/*"

Div / Box Sizing and Positioning

Divs are block elements and take up the entire width of a page. You can break up a div into two "columns" to present content in two separate dives to appear side-by-side instead of one on top of the other. To do this, add the rule "display: flex;" to the appropriate CSS to allow two divs to appear side by side

Code-Test-Refine: The Process of coding

Start coding to match mock up: add content, look for natural "boxes", add headings and tags. Test the code in the browser and then fix anything that needs to be fixed. Repeat the process until it's perfect.
